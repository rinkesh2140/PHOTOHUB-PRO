🚀 QUICK START (5 MINUTES)

Step 1: Set Up Database
────────────────────────
$ createdb photohub_pro
$ psql photohub_pro < database-schema.sql

Step 2: Configure Environment
─────────────────────────────
Create .env file:

DATABASE_URL=postgresql://user:password@localhost:5432/photohub_pro
JWT_SECRET=your-secret-key
FRONTEND_URL=http://localhost:5173

CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=your-key
CLOUDINARY_API_SECRET=your-secret

EMAIL_USER=your-gmail@gmail.com
EMAIL_PASSWORD=your-app-password

Step 3: Install Dependencies
────────────────────────────
$ npm install

Step 4: Start Backend
────────────────────
$ npm run dev

Step 5: Start Frontend
────────────────────
(In another terminal)
$ npm run dev

Step 6: Visit App
───────────────
http://localhost:5173

🎯 FEATURE IMPLEMENTATION DETAILS

FEATURE #1: SLIDESHOW WITH MUSIC
────────────────────────────────
Backend Logic:
  1. Photographer creates slideshow
  2. System fetches all project photos
  3. Creates slideshow record with music style
  4. Maps all photos to slideshow in order

Frontend Logic:
  1. Show music selection (4 styles)
  2. Auto-play photos with transitions
  3. Player controls (pause, speed, full-screen)
  4. Share link generation

Music Styles:
  • Romantic (piano + strings)
  • Upbeat (energetic)
  • Emotional (touching)
  • Classic (timeless)

FEATURE #2: PHOTO TIMELINE
──────────────────────────
Database Fields:
  • event_time_label: "Getting Ready", "Ceremony", etc
  • event_time_order: 1, 2, 3... for ordering

Photographer Workflow:
  1. Upload photos
  2. Tag each photo with event time
  3. Set order

Client Workflow:
  1. Open timeline
  2. See all events as collapsible sections
  3. Click to expand and see photos from that moment
  4. Share photos from specific events

Default Events:
  1. Getting Ready
  2. Pre-Ceremony
  3. Ceremony
  4. Cocktail Hour
  5. Reception
  6. Cake Cutting
  7. First Dance
  8. Send-off

FEATURE #3: SMART PEOPLE TAGGING
─────────────────────────────────
Workflow:
  1. Photographer reviews each photo
  2. Types in person names: "Bride", "Groom", "Mom", etc
  3. Saves tags

Client Benefit:
  1. Click "Bride" → see all photos with bride
  2. Click "Mom" → see all photos with mom
  3. Photo count per person
  4. Bulk download by person

Technical:
  • Store person_name in photo_tags table
  • Multiple people can be in one photo
  • Efficient indexing on person_name

FEATURE #4: INSTAGRAM STORY GENERATOR
──────────────────────────────────────
Workflow:
  1. Client selects photo
  2. Chooses template (Simple, Quote, Announcement)
  3. Adds text: "Just got married! 💕"
  4. Adds hashtags: #wedding #love
  5. Click "Generate"
  6. System creates 1080x1920 PNG
  7. Download and upload to Instagram

Templates:
  • Simple: Photo + gradient overlay
  • Quote: Large centered text + hashtags
  • Announcement: Bold text + high contrast

Preset Options:
  • Text: "Just got married!", "Best day ever!", etc
  • Hashtags: #wedding #love, etc (clickable)

FEATURE #5: ANNIVERSARY REMINDER
────────────────────────────────
Automated Process:
  1. Cron job runs daily at 00:00 (midnight)
  2. Finds projects with anniversary today
  3. Fetches top 5 photos (by likes/views)
  4. Sends beautiful HTML email
  5. Includes "Claim Free Photoshoot" button
  6. Updates anniversary_reminders table

Email Template:
  • Subject: "💕 One Year Ago Today! + Free Photoshoot"
  • Top 5 photos displayed
  • Claim button
  • 30-day offer expiry

FEATURE #6: FREE 2-HOUR ANNIVERSARY PHOTOSHOOT
───────────────────────────────────────────────
Client Workflow:
  1. Receives anniversary email
  2. Clicks "Claim Your Free Photoshoot"
  3. Selects preferred date (within 30 days)
  4. Submits claim

Photographer Workflow:
  1. Receives notification
  2. Checks availability
  3. Confirms date with client
  4. Shoots anniversary photos
  5. Uploads to same project
  6. Client sees "Anniversary Year 1" section

Business Value:
  • Repeat shoot = ₹15,000-20,000 revenue for photographer
  • Platform gets commission
  • Client keeps coming back
  • Lifetime value increases

❓ TROUBLESHOOTING

Anniversary Email Not Sending
  • Check email credentials in .env
  • Verify Gmail app password (not regular password)
  • Check cron job is running
  • Check database for anniversary_reminders

Slideshow Not Playing
  • Verify music URLs are valid
  • Check browser console for errors
  • Ensure photos are uploaded

People Tagging Not Working
  • Verify photos are uploaded
  • Check @google-cloud/vision credentials
  • May need to manually tag instead of auto-detect

Instagram Story Generation Failing
  • Check sharp library is installed
  • Verify photo URLs are accessible
  • Check image dimensions
